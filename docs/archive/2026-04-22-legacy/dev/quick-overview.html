🧠 01 — What Is This

Multi-tenant SaaS that gives every user a private, hosted AI co-worker. One subscription → dedicated AI agent with a custom subdomain, 25+ messaging channels, shared file workspace, deep third-party integrations, and a memory that never forgets anything.

TL;DR: Think "Google Drive + AI assistant + zero data exposure" — set up in 60 seconds, no DevOps required. Built on OpenClaw (337k ⭐ on GitHub).
🎯 02 — The Pitch

Problems we solve

  • AI forgets everything — ChatGPT/Claude reset every session. After 1 year of daily use, they know nothing more than day one.
  • No true privacy — Every hosted AI platform can read your conversations. We physically cannot.
  • No shared workspace — AI integrations are isolated silos. No single agent spans all your tools.
  • Setup is a barrier — OpenClaw has 337k stars but requires terminal + Docker + Node. Fewer than 5% of people who'd benefit can self-host it.

  • DAG-based memory that stores every message, forever
  • Shared file workspace user + agent both read and write
  • 25+ channels, 40+ integrations, 30+ pre-built skills
  • Managed hosting — zero DevOps for the user

🧬 05 — Memory Engine (Never Forget)

Layer 1 — Verbatim Recall: lossless-claw

Replaces OpenClaw's sliding-window truncation with a DAG summarization system. Every message stored in SQLite. Nothing ever deleted.

  • lcm_grep — full-text + regex search across all messages and summaries
  • lcm_expand_query — deep recall via sub-agent delegation, drills down to raw messages
  • lcm_describe — inspect specific DAG nodes or stored files by ID
  • Depth-aware summaries — leaf → condensed → arc-level. Nothing discarded.
  • Large file interception — files >25k tokens stored separately, retrievable on demand
Practical demo: March 2027 — "What was that contractor's name from January 2026?" → Agent searches DAG, expands summary back to original message, returns exact answer. Not an approximation.
⚠ Dependency: lossless-claw requires OpenClaw PR #22201 (not yet merged). Must fork + patch until upstream merges. Prioritize this.

Layer 2 — Semantic Recall: Qdrant

Self-hosted vector DB sidecar. Meaning-based retrieval — "Find that Montreal renovation project" works even if those exact words were never used.

  • Runs entirely in-container — no data leaves the instance
  • Cross-document search: conversations + workspace + knowledge base simultaneously
  • MCP tool bridge — OpenClaw queries it natively mid-conversation

Layer 3 — Structured Memory: Mem0

Self-hosted AI memory layer. Auto-extracts entities from conversations into a personal knowledge graph.

⚠️ 15 — Watch Out

Critical Issues

  • lossless-claw needs PR #22201 — not merged upstream yet. Fork + patch. Top priority before memory marketing.
  • Persistent volumes — must be mounted in Dokploy before lossless-claw ships. SQLite can't live in ephemeral container storage.
  • Stripe webhook — not tested end-to-end in production yet. Must validate before public launch.
  • DNS updates needed — *.nickybruno.com + dok.nestai.nickybruno.com must point to new IP 35.202.32.236

Known Gotchas

  • Next.js 15 + next-auth has prerender bug on error pages — build warnings are OK
  • AUTH_TRUST_HOST=true required for NextAuth in production
  • Middleware must be in src/middleware.ts not project root
  • Dokploy inline compose needs sourceType: "raw"
  • OpenClaw uses TELEGRAM_BOT_TOKEN + OPENAI_API_KEY env vars — not OPENCLAW_*
  • Brand: emerald accent once per screen max — never decorative
  • Layouts: Root layout has NO nav. Public pages use PublicNav. Dashboard uses DashboardHeader.
